# Frontend Patterns

## Layout Patterns

- dashboard with all key surfaces visible at once
- split-pane master-detail workflow
- tabbed application shell
- step-by-step wizard or sequential flow
- focused single-task view

## Navigation Patterns

- sidebar navigation for multi-area apps
- tab bar for peer sections
- stacked drill-in flow for list-to-detail transitions
- command palette or search overlay for power-user actions

## Selection Guidance

- Choose the pattern that best matches the primary user path, not the one with the most UI chrome.
- Prefer a shell that can actually support the sprint plan the spec promises.
